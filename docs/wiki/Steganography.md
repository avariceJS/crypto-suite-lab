# LSB-стеганография

Используется простейшая модификация младшего бита (Least Significant Bit) в
байтах пикселей RGBA-изображения.

## Формат полезной нагрузки
| Биты | Назначение |
|------|------------|
| 0 — 31 | длина сообщения L (uint32, LSB-first) |
| 32 — 32+8L | байты сообщения, по 1 биту в LSB каждого байта пикселей |

## Ёмкость
Для буфера в N байт пиксельных данных: `floor((N − 32) / 8)` байт сообщения.

## Реализация
[`pd::stego::embed`](../../core/include/steganography.h:1) /
[`pd::stego::extract`](../../core/include/steganography.h:1) работают на
`std::vector<uint8_t>` — клиент сам конвертирует `QImage` в RGBA8888 и
сохраняет результат в **PNG** (формат без потерь — обязательное условие LSB).

## Ограничения
- JPEG не поддерживается (потери разрушат биты).
- Изменения в LSB-канале статистически детектируемы — это учебный, а не криптостойкий метод.
