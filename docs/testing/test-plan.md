# Тест-план — PD Crypto Suite

## 1. Цели тестирования
Проверить, что реализация соответствует функциональным требованиям FR-01…FR-10 и
нефункциональным NFR-01…NFR-07.

## 2. Объекты тестирования
- core: Vigenere, SHA-384, Chord, Steganography
- server: команды протокола, БД, авторизация, роли
- client: GUI, диалог входа, вкладки функций, админ-панель

## 3. Стратегия
Многоуровневая:
1. **Unit-тесты** алгоритмов (QtTest) — автоматизация, выполняется через CTest.
2. **Интеграционные** — ручные сценарии «клиент → сервер → БД».
3. **Smoke/UI** — чек-лист на GUI.
4. **Безопасность (smoke)** — проверка, что неаутентифицированный клиент не может
   вызывать защищённые команды; user не может вызывать admin-команды.

## 4. Окружение
- ОС: Ubuntu 22.04 / macOS 14 / Windows 11
- Qt 6.6
- CMake 3.27
- Docker 24

## 5. Критерии входа
- Сборка проекта без ошибок.
- Сервер успешно стартует, БД создаётся.

## 6. Критерии выхода
- 100% unit-тестов проходят.
- Все P1/P2 дефекты закрыты.
- Чек-лист GUI пройден ≥ 95%.

## 7. Риски
| Риск | Митигация |
|------|-----------|
| Несовместимость Qt версий | фиксируем Qt6 в CMake |
| Поломка LSB при пересохранении в JPG | UI разрешает только PNG |
| Race conditions в БД | QMutex в Database |
